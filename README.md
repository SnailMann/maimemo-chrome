# maimemo-chrome

> 基于墨墨开放 API 的轻量 Chrome 扩展。  
> 在网页中选中英文单词后，可以一键加入墨墨云词本，并对当前云词本中的单词进行本地高亮。

`maimemo-chrome` 是一个面向英语阅读场景的第三方 Chrome 扩展，核心目标很简单：

- 把你在网页里看到的英文单词，快速加入当前墨墨云词本
- 把当前云词本全量同步到本地缓存，并在网页里轻量标红这些单词

插件刻意保持轻量，只围绕“加词”和“识词”两件事展开，不做复杂学习面板，也不在浏览网页时实时请求云端接口。

> 说明：本项目基于墨墨开放 API 构建，为第三方开源工具。

## 核心能力

- 右键选中文本后，可直接将单词加入当前云词本
- 双击选中文本后，弹出快捷操作面板
- 双击面板支持两个动作：
  - `添加单词`
  - `不再标记`
- 当前云词本会同步到本地缓存，网页高亮优先读取本地缓存
- `不再标记` 只影响本地网页高亮，不会修改云词本，也不会改动本地缓存词表
- 高亮样式只保留“文字变红”，尽量减少对原网页视觉的打扰

## 使用流程

1. 在设置页填写并保存墨墨开放 API Token
2. 自动或手动获取当前账号下的全部云词本
3. 选择一个词本作为“当前待同步云词本”，并设置同步周期
4. 保存后，插件会把该云词本全量同步到本地缓存
5. 之后浏览网页时，缓存中的单词会被轻量标红
6. 你可以通过右键或双击快捷面板，把新单词追加到当前云词本

## 安装

### Chrome 商店

Chrome Web Store 链接将在发布后补充到这里。

### 本地安装开发版

1. 打开 Chrome，进入 `chrome://extensions/`
2. 打开右上角“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择本项目根目录：

```text
maimemo-chrome/
```

5. 安装后点击“详情”或“扩展程序选项”，进入设置页完成 Token 和云词本配置

## 交互方式

### 1. 右键添加单词

选中网页中的一个英文单词，右键后点击 `加入墨墨词本`，即可将该单词追加到当前云词本。

### 2. 双击快捷面板

双击选中的英文单词后，会弹出一个纵向快捷面板：

- `添加单词`
  与右键菜单功能一致，将当前单词加入当前云词本
- `不再标记`
  将该单词加入本地“不标记黑名单”，后续不再在网页里高亮

### 3. 当前支持的选中文本

目前更适合处理“单个英文单词”，例如：

- `indie`
- `startup`
- `co-founder`
- `don't`

如果选中的是整句、多个单词或非英文内容，插件会忽略这次添加操作。

## 本地缓存与同步说明

插件不会在你每次打开网页时都去实时读取墨墨云端数据。

它的工作方式是：

1. 先把你选定的云词本全量同步到本地缓存
2. 浏览网页时，只使用本地缓存做高亮匹配
3. 后台按照你设置的同步周期，定时刷新这份本地缓存

这样做的好处是：

- 页面高亮更快
- 浏览时不依赖实时网络请求
- 云端 API 调用次数更少
- 更容易控制浏览器性能开销

## 权限与隐私

### 扩展权限用途

| 项目 | 用途 |
| --- | --- |
| `storage` | 保存 Token、当前词本、同步周期、本地缓存词表和本地不标记黑名单 |
| `alarms` | 按设定周期自动同步当前云词本到本地缓存 |
| `contextMenus` | 提供右键“加入墨墨词本”菜单 |
| `https://open.maimemo.com/*` | 访问墨墨开放 API，读取词本、更新词本、同步数据 |
| 内容脚本运行于 `<all_urls>` | 仅用于本地高亮网页单词，以及支持双击后弹出快捷操作面板 |

### 数据行为说明

- Token、同步设置、本地缓存词表和“不标记黑名单”保存在浏览器本地存储中
- 插件不会上传你的网页正文内容，也不会上传你的浏览历史
- 浏览网页时的高亮匹配在本地完成，不依赖实时云端请求
- 只有在以下场景中，插件才会请求墨墨开放 API：
  - 读取云词本列表
  - 获取当前云词本详情
  - 将选中的单词追加到当前云词本
  - 按同步周期刷新当前云词本到本地缓存

## 性能设计

这个项目从一开始就把“轻量”和“可持续使用”放在比较靠前的位置，当前实现主要遵循这些原则：

- 高亮基于本地缓存，不在浏览时实时拉取云端数据
- 内容脚本在 `document_idle` 时机执行，尽量不干扰首屏渲染
- 只处理可见文本节点，不直接用 `innerHTML` 粗暴替换整段 HTML
- 页面新增内容采用增量处理，避免反复全页重刷
- 高亮样式只保留文字变红，尽量减少额外布局和绘制开销

## 设置页逻辑

设置页按 4 个步骤组织：

1. 保存 Token
2. 获取全部云词本
3. 保存待同步云词本，并设置同步周期
4. 查看当前本地缓存词表

这样可以把“鉴权”“选词本”“同步缓存”“查看结果”四件事拆开，方便排查问题，也更利于后续维护。

## 常见问题

### 为什么需要访问所有网页？

因为插件需要在网页中执行两件本地动作：

- 识别并高亮本地缓存中的单词
- 支持双击选中文本后弹出快捷操作面板

这些动作都在本地完成，不会把整页内容上传到云端。

### “不再标记”会不会把单词从墨墨里删掉？

不会。  
`不再标记` 只会把单词加入本地黑名单，让它不再参与网页高亮。它不会修改墨墨云词本，也不会删除本地缓存词表中的原始数据。

### 为什么网页高亮不是实时和云端完全一致？

因为插件采用的是“云词本 -> 本地缓存 -> 网页高亮”这条链路。  
这样可以减少网络请求，提升页面体验，也更适合长时间常驻使用。
